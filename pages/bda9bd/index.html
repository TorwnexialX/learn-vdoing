            构造函数

            # 1. 基本定义

            构造函数的出现是为了解决类内成员(esp. 数据成员)初始化的问题 如先前的Clock类，每次实例化后都要设置数据成员初始值，而采用了构造函数后就可以简化该步骤，如，在Clock定义中加入下述代码

            public:
	Clock(int h, int m, int s){//与类同名
		Hour = h;//对数据成员设置初始值
		Minute = m;
		Second = s;
	}
	//上述定义也可以按照如下结构写
    Clock(int h, int m, int s):Hour(h), Minute(m), Second(s){};
            • 构造函数当且仅当在对象被创建时调用
            • C++中类的默认构造函数是一个空函数，什么也不做，如果用户在类中声明了构造函数，默认构造函数就不再起作用了
            • 构造函数没有返回值，名字与类名相同

            # 2.默认值

            public:
    Clock(int h = 3, int m = 5, int s = 1):Hour(h), Minute(m), Second(s){};
    以上形式说明，若对应位置不给构造函数传参，则采用上述默认值
    但是，默认值不能随意设置，只能从后往前连续进行
    
    如，下式是合理的
    Clock(int h, int m = 5, int s = 1):Hour(h), Minute(m), Second(s){};
    但，下式不合理，因为默认值只能倒序连续存在
    Clock(int h, int m = 5, int s):Hour(h), Minute(m), Second(s){};

            初始值的设置会对构造函数的重载有影响，见后续解释

            # 3. 给构造函数传参

            传给构造函数实参的两种方式： 构造函数仅一个参数： className obj = parameter; 构造函数有一个或多个参数： className obj(parameter list); [注]：第一个传参中的等号的意义不是赋值，而是给构造函数的参数 如

            假设构造函数三位均有默认值，则下列均合法
Clock aclock(3, 5, 1);原来有默认的位置被新设值覆盖
Clock aclock(5, 5);
Clock aclock = 3;若值不是三个，则从第一个开始顺序设置参数
假设后一位有默认值，则
Clock aclock = 1; 不合法
Clock aclock(3, 5); 合法
Clock aclock(3, 5, 1); 合法

            # 4. 重载构造函数

            重载构造函数：

            • 一个类可以提供多个构造函数，即构造函数的重载
            • 重载的目的是为了满足不同的初始化需要
            class Clock{
	public:
	Clock();
	Clock(int a, int b);
}
            • 不过，构造函数默认值的设置会影响构造函数的重载（易错点），如
            class Clock{
  private:
   int Hour,Minute, Second;
  public:
  Clock(int h = 0, int m = 0, int s = 0);
  Clock(char *timestr);
  ……
};

            该类就不能有Clock();形式的重载，因为其也可以理解为

            Clock(int h = 0, int m = 0, int s = 0);

            该函数均采用默认值的情况，有歧义，该点一定要注意
