    Vdoing上手

    # 1. 简介

    Vdoing是VuePress v1.x下的一个主题，作者创立这个主题的初衷是打造一个易用的知识管理体系，同时也可以作为简洁的博客搭建框架

    而其特性主要如下：

    1. 高效的网页配置：自动生成结构化侧边栏，front matter, 目录页, 面包屑，自动创建归档页，标签检索，简化了不少不必要的重复性劳动，提高了网页生成的效率
    2. 优秀的视觉效果：提供了四种不同的网页颜色模式，提供了易于阅读的目录收缩、左右快速换页，一键返回顶部等互动
    3. 丰富的博客功能：对文章提供了有关作者和时间的设置，有博主相关的信息栏，同时提供了分类检索，评论功能，方便浏览者快速找到对应文章，并留下反馈评论

    下面，我们将根据自己的经验介绍一下vdoing的入门

    # 2. 上手

    # 1. 从作者已有项目的基础上修改
    # clone the project
git clone https://github.com/xugaoyi/vuepress-theme-vdoing.git

# enter the project directory
cd vuepress-theme-vdoing

# install dependency 注意：如安装不成功请关闭淘宝源。
npm install # or yarn install

# develop
npm run dev # or yarn dev

    注意

    VuePress要求的node.js版本是大于v8.6，但是要使用Vdoing主题需要在v14.17以上，同时要小于等于v16.20.0

    # 2. 直接在vuepress项目的基础上使用vdoing主题
    #进入项目目录
npm init #yarn init
npm install -D vuepress #先准备vuepress
npm install -D vuepress-theme-vdoing

    此时，我们已经下载完了vuepress和vdoing主题，想要进行应用，需要在config.js中进行配置，其目录为/docs/.vuepress/config.js，配置内容如下

    module.exports = {
  theme: 'vdoing'
}

    注意

    不过仍有以下几点要注意：

    1. 为了调试，不要忘了在/package.json下添加调试相关代码，即
    "docs:dev": "vuepress dev docs",
"docs:build": "vuepress build docs"
    1. 为了使用vdoing相关特色，config.js中仅有上述配置的话并不能使用全，就比如vdoing经典的结构化侧边栏sidebar: structuring;等，所以建议参考vdoing官网有关主题配置的说明 (opens new window)或者直接参考作者在github官网提供的config.js示例模板 (opens new window)，作者的项目默认用的是config.ts，如果想用该格式配置的话也可参考作者在官网提供的config.ts示例模板 (opens new window)。
    2. 使用淘宝源安装可能会出现问题，如果你在dev过程中出现了以下报错

    提示你缺少vue-template-compiler，而实测中，即便安装上该模块，也不成功，但不过该问题在关闭淘宝源后得到解决，所以如果安装过程中出现类似报错，可以考虑可能是由于淘宝源造成的问题

    # 3. 有关两种方式的选择

    vdoing官网中，作者建议直接在其原本项目的基础上修改，不过实际测试中，我们发现在作者原有项目的基础上修改成本较高，所以当你的整体结构和内容变化较大时，还是建议在vuepress的基础上使用vdoing主题。所以针对上述两种方式的优缺点，我们做了以下总结

    应用作者原有项目 自己添加主题
    优点 极少在配置方面出现问题，一般不会因为环境等报错 编辑自由度高，能按照自己的需求个性化地搭建博客，同时配置上可以依据需求添加，较为简洁
    缺点 原有配置繁杂，不够简明，当想要找对应配置时很难找，而且当对内容大规模修改时，需要牵连改动的地方非常多，配置粘性大 在基础的环境搭建上可能需要花不少时间，而且配置上也要从零写起，工作量巨大
    适用情况 想要搭建知识博客，且以搭建知识体系为重，不太注重个性化，且不想处理太多环境搭建上的问题 想要搭建个性化的，有鲜明特色的博客，且肯投入时间研究环境配置问题

    注：以上提到的配置成本高是针对初学者而言，对于有一定前端基础的用户可以忽略
